# Gateway-tools UI Audit

本文档固化 Gateway-tools 的 UI/UX 方向、设计 token、组件密度和响应式规则。后续新增页面或重构现有页面时，应优先遵守本文档，再结合具体业务页面做局部例外。

## 产品定位

Gateway-tools 是面向开发者和高频账号运维场景的桌面生产力工具。界面应服务于快速扫描、稳定操作和低认知负担，而不是营销页式展示。

推荐方向：

- 高密度但有秩序的操作型仪表盘。
- 清晰的导航层级、状态反馈和批量操作入口。
- 稳定的 hover/focus 状态，不通过缩放造成布局跳动。
- 兼顾浅色和深色主题，可读性优先于装饰性。

避免方向：

- 大面积玻璃拟态、过大的圆角和过重阴影。
- 卡片套卡片、首屏信息过稀、装饰性渐变过多。
- 只靠颜色表达状态。
- 缺少键盘焦点样式的 icon-only 控件。

## Design Tokens

基础 token 定义在 `src/styles/base.css`。

### Radius

工具型界面使用紧凑圆角：

- `--radius-sm: 6px`：徽标、小按钮、表格内控件。
- `--radius-md: 8px`：输入框、常规按钮、列表项。
- `--radius-lg: 10px`：工具栏、卡片、表格容器。
- `--radius-xl: 14px`：浮层、较大的面板。
- `--radius-modal: 14px`：常规模态；大型沉浸模态最多 16px。
- `--radius-full` 仅用于圆形图标按钮、pill、开关。

硬性约束：

- 工具区、卡片、表格建议 6-10px。
- 模态和弹层建议 12-16px。
- 不为常规业务卡片使用 20px 以上圆角。

### Color

当前主色为蓝色 `--primary`，辅助色为青绿色 `--accent`。状态色统一使用 `--success`、`--warning`、`--warning-strong`、`--danger`。

状态表达应同时包含文字、图标或数值，不只依赖颜色。

### Focus

所有可交互元素必须有 `:focus-visible`：

- 使用 `--focus-ring` 作为外圈。
- 使用 `--focus-ring-soft` 作为扩散层。
- 不允许只写 `outline: none` 而没有替代焦点样式。

## Component Density

### Dashboard

Dashboard 是操作入口，不是展示页。卡片应更像信息面板：

- 统计卡高度保持在约 56-72px。
- 平台卡首屏应优先展示平台名、刷新/隐藏操作、当前账号、推荐账号和主要配额。
- 当前/推荐区应保持横向可扫描；内容不足时使用紧凑空状态。
- 配额列表应限制高度并允许内部滚动，避免单个卡片撑高整行。
- Footer 操作使用整行轻量按钮，不做过强视觉主按钮。

### Cards And Tables

- 卡片 padding 优先 10-14px。
- 表格容器使用 8-10px 圆角，表格行 hover 只改变背景或边框。
- 高频页面优先提供 compact/list 视图，不依赖大卡片作为唯一布局。

### Motion

- Hover 使用背景、边框、颜色或阴影反馈。
- 避免 `transform: scale(...)` 用在常规按钮、卡片、表格行和导航项。
- 彩蛋、游戏或一次性庆祝动效可以例外，但不应影响核心导航。
- 动效时长建议 120-200ms；复杂进度动效应尊重 `prefers-reduced-motion`。

## Responsive Rules

### Desktop

- 默认使用左侧导航 + 主内容区。
- 宽屏页面保持内容密度，不将卡片无限拉宽。
- 经典侧边栏用于平台较多或需要文字导航的场景。

### 900px 以下

窄屏进入底部导航模式：

- 侧边导航变为底部工具栏。
- 桌面主窗口 `minWidth` 必须低于该断点；当前 Tauri 主窗口最小宽度为 720px，以便真实触发 900px 以下布局。
- 用户保存的经典侧栏偏好只在桌面宽度生效；900px 以下统一压缩为底部工具栏。
- 主内容取消左侧预留，底部增加安全留白。
- 导航图标横向滚动，当前项高亮。
- 浮动日志按钮隐藏或并入底部导航/更多菜单。
- 弹出菜单从底部导航上方展开，避免遮挡内容。

### 640px 以下

- 工具栏纵向堆叠，主要操作靠右或换行。
- Dashboard 平台卡单列展示。
- 不出现页面级横向滚动，局部表格可横向滚动。

## Accessibility Checklist

- 按钮、输入框、选择器、卡片按钮、下拉触发器都具备 `:focus-visible`。
- icon-only 按钮必须有 `aria-label` 或 `title`。
- 错误消息使用文字说明；重要错误区域应使用 `role="alert"` 或 `aria-live`。
- 可点击元素使用真实 `button` 或 `a`，避免只有 `div onClick`。
- 颜色不是唯一的信息载体。
- `prefers-reduced-motion` 下禁用非必要动画和 transform。

## Current Follow-up Areas

- 将更多账号页的大卡片圆角收敛到 token。
- 给所有自定义下拉和弹层补齐键盘导航。
- 逐步把重复 CSS 收敛为共享组件层，减少各平台页面视觉漂移。
